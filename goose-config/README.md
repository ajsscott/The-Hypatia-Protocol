# goose-config/

Hypatia's Goose custom-distro configuration. Loaded by Goose when launched against this repo as `HYPATIA_REPO_ROOT`.

## Install Goose (if not present)

```bash
brew install block-goose       # Homebrew (if available)
# OR download from https://github.com/block/goose/releases
```

Verify:
```bash
goose --version
```

## Initial Hypatia setup

```bash
# From repo root
cd /Users/ajsscott/GitHub/The-Hypatia-Protocol

# Build the Hypatia MCP servers
cargo build --release

# Point Goose at this repo's config
export HYPATIA_REPO_ROOT="$PWD"
export GOOSE_CONFIG_PATH="$PWD/goose-config/config.yaml"

# First run — Goose initializes provider config interactively
goose configure
# Pick Ollama; URL http://127.0.0.1:11434; model TBD (Q-17 re-evaluation)

# Verify Hypatia loads
goose session start --name hypatia-test
# In the session: "Hello, Hypatia." should produce Alexandrian-register reply
```

## Files in this directory

| File | What it does |
|---|---|
| `config.yaml` | Main Goose distro config — provider, extensions, system prompt source, defaults |
| `system-prompt.md` | The compact kernel concatenated for Goose's system-prompt slot. Regenerated by `regen-system-prompt.sh`. |
| `regen-system-prompt.sh` | Build script: concatenates `kernel/01-04.md` → `system-prompt.md` for Goose to load |
| `extensions.yaml` | MCP server registrations (Hypatia's protocols server + future vault-rw, save-session) |
| `goosehints.md` | Optional per-session hints for Goose |

## Caveats — verify against actual Goose schema

This config is **structured based on Goose's documented patterns as of 2026-04**. The exact YAML schema may differ slightly. After installing Goose, run `goose configure` and inspect `~/.config/goose/config.yaml` (or platform-equivalent) to see Goose's expected field names. Update this file to match.

If a field name in `config.yaml` here doesn't resolve, the most likely fixes:
- `providers.ollama.base_url` ↔ `providers.ollama.endpoint` ↔ `providers.ollama.host` (schema varies)
- `extensions.<name>.command` ↔ `extensions.<name>.cmd` (varies)
- `system_prompt` ↔ `system` ↔ `instructions` (varies)
